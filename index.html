Manus 和 Claude Code 能做的，OpenClaw 也能做 —— 而且是开源的。但我不推荐你现在用它。

为什么 OpenClaw 会火？

顶尖模型的智能水平发展到了可以支撑 Agent 运转的程度

2025 年以来，Claude、GPT-4o、Gemini 等顶尖模型在工具调用、多步推理、长上下文理解等关键能力上突破了临界点，使得 Agent 不再只是 Demo，而是真正能在复杂场景中稳定运行的产品。OpenClaw 的爆发，本质上是站在了模型能力跨过 Agent 可用门槛的时间点上。

IM 集成

直接在你常用的聊天工具里使用 AI Agent：WhatsApp、飞书、钉钉、Slack、微信（通过插件）……不需要单独打开一个 App，对话即操作。

定时 & 主动任务

通过 Heartbeat 机制定时"醒来"检查任务，通过 cron 目录定义日报、周报等定时自动执行的任务。不需要你主动发起，Agent 会自己干活。

类 Agent 能力

自主调用工具、多步推理、记忆系统 —— 不只是聊天，而是真正能帮你完成任务的智能体。Manus 和 Claude Code 能做的事，OpenClaw 也能做。

OpenClaw 工作空间

openclaw-workspace/
(session memory)

六层记忆系统

OpenClaw 用六层文件来定义 Agent 的"记忆"，从内到外，从稳定到临时：

SOUL
SOUL.md — 不可变的人格内核，定义 Agent「是谁」，不应被后续对话修改。
AGENTS.md — 身份定义、行为边界、回复风格，相当于 system prompt 的文件化版本。
TOOLS.md — 工具文档：我能用什么工具？怎么用？
USER.md — 用户偏好：称呼、偏好、关系，逐渐积累。
MEMORY.md — 长期记忆，Agent 在对话中主动写入的持久化事实和决策。
Session — 会话记忆：当前对话的上下文，对话结束即清除。

Heartbeat & Cron

OpenClaw 不只是被动回答问题。它能主动"醒来"执行任务。

HEARTBEAT.md — 主动检查

定义 Agent 什么时候主动联系你。比如每 4 小时检查一次待办事项，有紧急邮件时立即通知。

Skills 技能系统

Skills 是 OpenClaw 的能力扩展机制 —— 通过 Markdown 文件定义 Agent 能做什么、什么时候触发。

三层优先级

Workspace 级别当前项目专属，最高优先
User 级别用户自定义，跨项目通用
Bundled 级别内置技能，兜底

ClawHub 技能市场

社区贡献了 13,700+ 技能，一键安装，涵盖开发、写作、数据分析等各个领域。

Mac mini 家庭服务器

推荐 — 完全掌控，长期在线

1安装：npm install -g openclaw@latest
2初始化：openclaw onboard --install-daemon
3菜单栏应用：一键启停 Gateway，查看 channel 状态，系统通知
4远程访问：Tailscale Serve，手机/平板随时随地使用

Mac mini 低功耗、安静、常开，是完美的个人 AI 服务器。launchd 守护进程确保开机自动运行。成本：仅 API 费用。

避坑指南

OpenClaw 很酷，但现在入坑要三思。

安全风险

安全，安全，还是安全

  • CVE-2026-25253：远程代码执行漏洞，攻击者可通过恶意 Skill 执行任意命令
  • ClawHavoc 供应链攻击：ClawHub 上的恶意技能伪装成热门工具，窃取用户数据。Wiz 审计发现 ClawHub 上约 1/5 的 Skill 包含恶意代码
  • Meta AI 安全总监邮件被删事件：Meta 超级智能实验室对齐方向负责人 Summer Yue 让 OpenClaw 整理邮箱，Agent 无视「操作前先确认」的指令，批量删除了数百封邮件。她试图通过手机发消息叫停，Agent 反而加速「speed run」删除。最终她冲到电脑前拔掉了网线才制止。事后 Meta 禁止员工在公司设备上使用 OpenClaw
  • 密钥泄露：Moltbook 数据库配置不当，泄露 150 万条 API Key；Wiz 扫描发现 65% 的头部 AI 公司在 GitHub 上暴露了有效密钥；vibe coding 平台上 5600+ 个应用泄露了 400+ 个 API Key 和 175 条用户隐私信息（含医疗记录、银行账号）
  • AI Agent 删除硬盘内容：Google Antigravity IDE 用户让 AI 清理缓存，结果执行了 rmdir /s /q d: 删除了整个 D 盘，无法恢复；Replit AI Agent 删除了客户的生产数据库后试图用假数据掩盖；一位用户让 Claude Code 执行任务，结果 rm -rf 了整个 home 目录，多年家庭照片和工作项目全部丢失；Anthropic 的 Claude Cowork 在帮用户整理桌面时误删了 15 年共 1.5~2.7 万张家庭照片
  • 安全设计存在根本性缺陷：SecurityScorecard 发现 13.5 万个 OpenClaw 实例以不安全的默认配置暴露在公网上，已披露 9 个 CVE，其中 3 个有公开利用代码
成本问题

好模型贵，便宜模型性价比低

  • 效果好的模型（Claude、GPT-4）费用高昂，日常使用成本不低
  • 便宜模型需要花大量时间调教和修复，精力性价比很低
  • 建议：等半年，国内便宜模型追上当前第一梯队的 80% 再尝试
成熟度

思路好，但实现粗糙

  • OpenClaw 的架构设计理念确实领先，但代码实现质量参差不齐
  • 当前各大厂出的都只是 OpenClaw 的套壳版本
  • 建议：等半年，让大厂根据 OpenClaw 的思路开发出更稳定、易用的版本

Prompt 方法论

写好 Prompt 是用好 AI 的基础。

01

明确角色和目标

模糊
帮我写个总结
清晰
你是一位技术博客编辑。请将以下会议纪要总结成 3 个要点，每个不超过 50 字，面向开发者读者。
02

提供充分的上下文

缺少上下文
这个 bug 怎么修？
上下文充分
我在用 Python 3.11 + FastAPI。调用 /api/users 时返回 422 错误。请求体是 {"name": "test"}，模型定义如下：[代码]。帮我找出原因。
03

指定输出格式

无格式要求
分析这些数据
格式明确
请以 Markdown 表格输出，列包括：指标名称、本周数值、环比变化、趋势判断（上升/下降/持平）。

Skill 方法论

学会写 Skill，就是学会教 Agent 做事。

# 一个完整的 Skill 文件结构

---
name: weekly-report
trigger: cron("0 17 * * 5")  # 每周五 17:00
description: 生成本周工作周报
---

## 任务

1. 收集本周的 Git commit 记录
2. 汇总已完成和进行中的任务
3. 整理成周报格式，包含：
   - 本周完成的工作
   - 遇到的问题
   - 下周计划

## 输出要求

- 格式：Markdown
- 长度：300-500 字
- 发送到：飞书群 "技术周报"
1 触发条件 — 定义 Skill 的执行时机。支持 cron 表达式、关键词触发、事件触发等。
2 执行指令 — Skill 的核心。清晰地告诉 Agent 要做什么，步骤越具体越好。
3 输出要求 — 约束最终产出的格式、长度和去向，避免 Agent 自由发挥。
