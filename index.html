OpenClaw 工作空间

📂 openclaw-workspace/
💭 (session memory)

四层记忆系统

OpenClaw 用四层文件来定义 Agent 的"记忆"，从内到外，从稳定到临时：

SOUL
SOUL.md — 身份与人格：我是谁？我的行为准则是什么？几乎不变。
TOOLS.md — 工具文档：我能用什么工具？怎么用？
USER.md — 用户偏好：用户喜欢什么风格？有什么习惯？逐渐积累。
Session — 会话记忆：当前对话的上下文，对话结束即清除。

Heartbeat & Cron

OpenClaw 不只是被动回答问题。它能主动"醒来"执行任务。

HEARTBEAT.md — 主动检查

定义 Agent 什么时候主动联系你。比如每 4 小时检查一次待办事项，有紧急邮件时立即通知。

Skills 技能系统

Skills 是 OpenClaw 的能力扩展机制 —— 通过 Markdown 文件定义 Agent 能做什么、什么时候触发。

👤 User 级别用户自定义，跨项目通用
📦 Bundled 级别内置技能，兜底

避坑指南

OpenClaw 很酷，但现在入坑要三思。

安全风险

安全，安全，还是安全

▾
  • CVE-2026-25253：远程代码执行漏洞，攻击者可通过恶意 Skill 执行任意命令
  • ClawHavoc 供应链攻击：ClawHub 上的恶意技能伪装成热门工具，窃取用户数据
  • 安全设计存在根本性缺陷，社区已有多起安全事故
成本问题

好模型贵，便宜模型性价比低

▾
  • 效果好的模型（Claude、GPT-4）费用高昂，日常使用成本不低
  • 便宜模型需要花大量时间调教和修复，精力性价比很低
  • 建议：等半年，国内便宜模型追上当前第一梯队的 80% 再尝试
成熟度

思路好，但实现粗糙

▾
  • OpenClaw 的架构设计理念确实领先，但代码实现质量参差不齐
  • 当前各大厂出的都只是 OpenClaw 的套壳版本
  • 建议：等半年，让大厂根据 OpenClaw 的思路开发出更稳定、易用的版本

Prompt 方法论

写好 Prompt 是用好 AI 的基础。

01

明确角色和目标

❌ 模糊
帮我写个总结
✅ 清晰
你是一位技术博客编辑。请将以下会议纪要总结成 3 个要点，每个不超过 50 字，面向开发者读者。
02

提供充分的上下文

❌ 缺少上下文
这个 bug 怎么修？
✅ 上下文充分
我在用 Python 3.11 + FastAPI。调用 /api/users 时返回 422 错误。请求体是 {"name": "test"}，模型定义如下：[代码]。帮我找出原因。
03

指定输出格式

❌ 无格式要求
分析这些数据
✅ 格式明确
请以 Markdown 表格输出，列包括：指标名称、本周数值、环比变化、趋势判断（上升/下降/持平）。

Skill 方法论

学会写 Skill，就是学会教 Agent 做事。

# 一个完整的 Skill 文件结构

---
name: weekly-report
trigger: cron("0 17 * * 5")  # 每周五 17:00
description: 生成本周工作周报
---

## 任务

1. 收集本周的 Git commit 记录
2. 汇总已完成和进行中的任务
3. 整理成周报格式，包含：
   - 本周完成的工作
   - 遇到的问题
   - 下周计划

## 输出要求

- 格式：Markdown
- 长度：300-500 字
- 发送到：飞书群 "技术周报"
1 触发条件 — 定义 Skill 的执行时机。支持 cron 表达式、关键词触发、事件触发等。
2 执行指令 — Skill 的核心。清晰地告诉 Agent 要做什么，步骤越具体越好。
3 输出要求 — 约束最终产出的格式、长度和去向，避免 Agent 自由发挥。
